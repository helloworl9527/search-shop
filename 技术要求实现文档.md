## Plan: 多站点商品全量抓取与AI归一比价

目标是做一个独立新服务（不复用当前仓库模块），按你要求分三步严格门禁：先抓全、再归一、后输出；每一步都可测试且必须经你确认后进入下一步。  
推荐技术路线：Python（FastAPI + Playwright）+ 关系型数据库；原因是抓取与AI生态成熟、开发迭代快。

**Steps**
1. 阶段0：搭建最小可用骨架（依赖：无）
   - 建立独立服务端与测试页面。
   - 页面先支持：输入多个站点、启动任务、查看实时日志和抓取进度。
   - 定义统一数据结构：站点、分类页、分页页、原始商品、归一商品、价格来源。
   - 交付标准：你能在页面发起任务并看到任务状态。

2. 阶段1：全量抓取系统（依赖：1）
   - 分类发现：从入口页自动收集所有商品分类链接。
   - 分页发现：识别页码、next、查询参数翻页，防止漏页与死循环。
   - 商品提取：列表抽取商品卡片，必要时进入详情补齐信息。
   - 动态页面：静态抓取失败时自动切换浏览器渲染抓取。
   - 反爬异常：登录、验证码、挑战页统一记录并跳过，进入人工复查列表（符合你设定）。
   - 交付标准：多分类、多分页可完整抓取，不只单页面。

3. 阶段1测试与验收（依赖：2）
   - 指标：分类覆盖率、分页覆盖率、商品去重后总量、失败清单完整性。
   - 人工抽检：每个分类抽样核对“页面可见商品”是否都在结果中。
   - 通过条件（建议）：覆盖率不低于 98%，且你确认“无明显遗漏”。
   - 门禁：你确认后才进入阶段2。

4. 阶段2：商品归一分类（依赖：3）
   - 规则层（优先）：清洗文本、同义词词典、品牌词映射、噪声词移除。
   - AI层（兜底）：规则低置信样本交给语义模型判断是否同品并生成标准名。
   - 结果可解释：每条归一结果附置信度与判定来源（规则/AI）。
   - 示例目标：将“ChatGPT-Plus自助续费充值卡密”与“GPT自助充值卡密～支持Plus新开续费”统一到同一标准名。
   - 交付标准：跨站同类商品可聚合到同一标准商品名。

5. 阶段2测试与验收（依赖：4）
   - 构建同品/非同品测试集，评估准确率、召回率、误归类率。
   - 页面提供“低置信样本人工纠正”，纠正后可回写词典。
   - 通过条件（建议）：准确率不低于 90%，你抽检通过。
   - 门禁：你确认后才进入阶段3。

6. 阶段3：结构化输出与最低价排序（依赖：5）
   - 聚合同一标准商品下的全部来源记录。
   - 价格标准化（币种、符号、文本价格清洗）。
   - 按价格升序输出，最低价优先，保留来源链接与时间。
   - 支持页面查看 + JSON/CSV 导出。
   - 交付标准：重复商品能按最低价正确排序并展示全部来源。

7. 阶段3测试与最终验收（依赖：6）
   - 端到端：多站输入 → 全量抓取 → 归一分类 → 最低价排序 → 导出。
   - 一致性：页面结果与导出文件一致。
   - 你最终确认后，形成可持续迭代版本（增加新站点、扩展词典、提升模型）。

**Relevant files**
- 独立新项目（与现有仓库代码隔离）
- 服务端模块：任务调度、抓取引擎、归一服务、导出服务
- 测试页面模块：任务提交、结果查看、低置信纠正、导出
- 测试模块：单元测试、集成测试、端到端测试
- 验收文档：每阶段指标与人工抽检记录

**Verification**
1. 阶段1验证：覆盖率、抽检一致性、失败记录完整性。
2. 阶段2验证：归一准确率、误归类率、人工纠正回流效果。
3. 阶段3验证：最低价排序正确率、来源完整率、导出一致性。

**Decisions**
- 已按你的要求采用严格分阶段门禁。
- 抓取范围限定为商品列表与详情页，不含评论/博客等非商品内容。
- 登录/验证码场景当前策略是记录并跳过，不做自动绕过。
- 归一采用“词典规则优先 + AI语义兜底”。
- 输出以“标准商品名 + 全来源价格（最低价优先）”为核心。
